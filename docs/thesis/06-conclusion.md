# 第六章 总结与展望

## 6.1 工作总结

本文设计并实现了一套面向中国市场的咖啡馆发现与评价平台 SipSpot，采用 MERN 全栈架构（MongoDB、Express.js、React、Node.js），在传统 Web 应用基础上融合了多项人工智能能力，形成了具有一定工程实践价值的综合系统。

**系统功能层面**，本系统完整实现了咖啡馆信息的增删改查（CRUD）及 Cloudinary 云端图片上传、基于高德地图的地理位置检索与附近搜索、多维度评分体系（口味、价格、环境、服务、办公适合度，支持 0.5 步长的半星评分）、用户收藏夹与个人中心等核心功能，形成了闭环的用户使用流程。

**语义搜索模块**，本系统集成 multilingual-e5-base 本地推理模型，生成 768 维句向量并持久化至 MongoDB。用户可使用"安静适合写代码""适合拍照打卡"等自然语言表达检索意图，系统在应用层通过余弦相似度排序返回语义相关结果，支持中英文混合查询。针对模型未就绪的情形，设计了基于关键词匹配的降级机制，保障基础搜索功能的连续可用性。

**情感分析模块**，本系统接入阿里巴巴通义千问 API（qwen-plus），对用户评论进行自动化情感分析，输出情感极性（sentiment）、关键词（keywords）、摘要（summary）及置信度（confidence）四项结构化字段，并汇总生成咖啡馆级别的情感统计摘要。同时，本系统实现了基于本地词典的降级分析方案，在 API 不可用时仍可提供有限的情感判断能力。

**个性化推荐引擎**，本系统设计并实现了双路径推荐架构：当用户存在偏好向量时，采用向量余弦相似度对候选咖啡馆排序；当用户无历史行为时，退回规则路径，依据评分、距离、口味偏好等特征进行加权评分。用户偏好向量通过指数衰减加权平均（衰减系数 0.85，滑动窗口 30 条）从评论和收藏行为中自动学习更新，并引入探索推荐机制，避免推荐结果同质化。

综上，本系统在工程实现上将语义向量搜索、大语言模型情感分析与行为驱动向量推荐三大 AI 能力有机整合至 MERN 全栈架构中，验证了轻量级 AI 增强在中小型 Web 应用中的可行性。

## 6.2 不足与展望

### 当前不足

尽管本系统实现了预期功能，仍存在若干明显不足。

首先，**测试覆盖率为零**。项目已配置 Jest 与 Supertest 测试框架，但目前尚无任何测试文件。核心 AI 服务（语义搜索、情感分析、推荐引擎）均缺乏单元测试与集成测试保障，系统的稳定性依赖人工验证，存在回归风险。

其次，**应用层向量排序存在性能瓶颈**。当前余弦相似度计算在 Node.js 应用层逐条执行，时间复杂度为 O(n)，随咖啡馆数量线性增长，不适用于大规模数据场景。

第三，**推荐冷启动问题**。新用户无任何行为历史，系统只能退回规则路径推荐，推荐质量和个性化程度有限，影响新用户的初次体验。

第四，**数据枚举不一致**。推荐服务中 amenityBoost 参数采用英文枚举，而 Cafe 模型的 amenities 字段为中文枚举（如"电源插座"），两者不一致导致口味匹配逻辑存在静默错误，为潜在的数据质量隐患。

### 未来展望

针对上述不足，后续工作可从以下方向展开。

**向量数据库迁移**：将 embedding 字段从 MongoDB 普通字段迁移至 MongoDB Atlas Vector Search 或 Milvus 向量数据库，利用 HNSW 等近似最近邻（ANN）索引实现十万级以上数据的毫秒级检索，从根本上解决应用层排序的性能瓶颈。

**RAG 增强搜索**：结合检索增强生成（Retrieval-Augmented Generation）技术，在语义检索结果基础上引入大语言模型生成自然语言摘要，提升搜索结果的可解释性与用户可读性。

**多模态搜索**：引入图像 embedding 模型，对咖啡馆上传图片生成视觉向量，支持用户通过上传参考图片检索视觉风格相似的咖啡馆，拓展搜索维度。

**自动化测试体系建设**：依托已配置的 Jest + Supertest 框架，系统性地补充 AI 服务单元测试、API 路由集成测试及推荐引擎回归测试，建立持续集成流水线，提升系统长期可维护性与代码质量保障水平。
